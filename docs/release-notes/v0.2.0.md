# v0.2.0 — first tagged release

A Code-Mode MCP server for the entire UniFi ecosystem: Network, Site
Manager, and Protect. Five sandbox surfaces, two MCP tools, one QuickJS
WASM sandbox. Cloudflare's Code-Mode pattern applied to UniFi's 70+
endpoints — your LLM context stays small no matter how big the API gets.

Single-user (env vars) and multi-user (per-request HTTP headers) modes.
Stdio + Streamable HTTP transports. Cloudflare Workers entry. Strict TLS
by default with per-tenant overrides.

## What you can do

```js
// One key, every console you manage. No controller exposure.
const net = unifi.cloud.network('CONSOLE_ID');
const sites = net.sites.listSites();

// Same shape, Protect.
const p = unifi.cloud.protect('CONSOLE_ID');
const cameras = p.request({ method: 'GET', path: '/v1/cameras' });

// Or stay on the LAN.
const local = unifi.local.devices.listAdoptedDevices({ siteId: 'default' });
```

The full surface is documented in [`SKILL.md`](../../SKILL.md) (for the
LLM) and [`docs/usage.md`](../usage.md) (for humans).

## Highlights since the initial scaffold

- **Two new sandbox surfaces:** `unifi.local.protect.*` and
  `unifi.cloud.protect(consoleId).*`. Loader auto-fetches Ubiquiti's
  official Protect OpenAPI spec; bundled curated fallback (~18
  JSON-over-HTTP ops) ships as a CDN-unreachable backstop
- **Smarter `synthesizeOperationId`** — the official Protect spec ships
  every operation with `operationId: null`. The synthesizer recognises
  five common REST shapes and produces friendly names like `listCameras`,
  `cameraPtzPatrolStart`, `getCameraSnapshot` instead of the legacy
  `getV1CamerasIdPtzPatrolStartSlot` form
- **Three live bug fixes** caught by the cloud-Protect verification:
  - cloud-Protect proxy was gated on the Site Manager spec being
    loaded; the two specs are independent — fixed
  - Protect version-discovery probed the wrong URL for cloud baseUrls
    — fixed (skipped for `api.ui.com`, falls through to known versions)
  - `UnknownOperationError` attributed every miss to `unifi.local spec`
    regardless of which surface called it — fixed
- **Documentation correction:** the initial Protect implementation
  documented the cloud-Protect path as "UNVERIFIED — Ubiquiti has not
  publicly documented" the connector proxying Protect. That was wrong:
  `developer.ui.com/protect/v7.0.107/gettingstarted` exposes a
  "Remote" / "Local" base-URL selector that maps every operation to
  exactly the paths this server emits. All references corrected.

## Install

```bash
git clone https://github.com/jmpijll/unifi-code-mode-mcp.git
cd unifi-code-mode-mcp
npm install
cp .env.example .env  # set UNIFI_LOCAL_API_KEY and/or UNIFI_CLOUD_API_KEY
npm run build
npm start
```

For the multi-tenant HTTP transport, `MCP_TRANSPORT=http npm start` and
pass `X-Unifi-*` headers per request.
