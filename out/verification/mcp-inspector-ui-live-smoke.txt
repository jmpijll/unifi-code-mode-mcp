WHAT THIS PROVES
  ✓ The official MCP Inspector UI mode (the browser-based debugger,
    not just `--cli`) loads our server cleanly. The form fills with
    `node` + `/path/to/dist/index.js` + UNIFI_LOCAL_* env vars
    forwarded automatically from the launch command (-e flags).
  ✓ Clicking "Connect" succeeds — Inspector shows green
    "Connected", and the History pane records `1. initialize`.
  ✓ Clicking "List Tools" returns both `search` and `execute` with
    full descriptors. Inspector renders the tool descriptions
    correctly in the right-hand panel.
  ✓ Selecting `execute` reveals a populated form:
      - `code *` textarea with placeholder
        "JavaScript code to execute against the live UniFi APIs.
         Use await — operations are async."
      - "Run as task" checkbox
      - "Run Tool" button
  ✓ Pasting `var s = unifi.local.callOperation('getSiteOverviewPage');
    ({ siteCount: s.data.length, names: s.data.map(function(x){return x.name;}) });`
    into the code field and clicking "Run Tool" returns
    "Tool Result: Success" and the response object {siteCount: 1,
    names: [...]}. The TLS-insecure warning surfaces in the result
    body just like in CLI mode.
  ✓ History pane records `3. tools/call` after the execute, alongside
    the prior `2. tools/list` and `1. initialize`. Server Notifications
    pane records the corresponding stderr/log notifications from the
    server.
  ✓ Every visible UI affordance round-trips JSON-RPC over the
    stdio-bridged proxy correctly. The UI is the same protocol as
    CLI; the artefact closes the gap that previously read
    "Inspector UI mode unverified".

WHAT THIS DOES NOT PROVE
  ✗ HTTP / SSE transports inside the Inspector UI. We verified stdio
    only (which is the default and most common transport). The UI's
    Transport Type combobox offers SSE/HTTP, but those code paths
    are not exercised here.
  ✗ Long-running soak — the smoke is a single tools/list + single
    tools/call. Memory or session-stability under sustained UI use
    is unverified.
  ✗ Cross-platform browser variation. We drove the UI through
    Cursor's headless cursor-ide-browser MCP. Real users on Safari
    / Firefox / Edge / Chrome are unverified.
  ✗ Inspector v0.21.x. Pinned to v0.20.0 because v0.21.2 has the
    known missing-`commander` dependency error on Node v25 (see
    `out/verification/mcp-inspector-live-smoke.txt` for context).
