# v0.2.0-beta.1 — public beta

> **This is a beta. Not published to npm. Install from source.**
> Five sandbox surfaces are wired and tested against an in-process mock
> controller. Two surfaces (`unifi.cloud.network()` and
> `unifi.cloud.protect()`) are also verified live against the
> maintainer's home UDM-Pro. Other surfaces and other agent platforms
> are wired but unverified. See [`README.md`](../../README.md) and
> [`CONTRIBUTING.md`](../../CONTRIBUTING.md) for how to participate.

A Code-Mode MCP server for the entire UniFi ecosystem: Network, Site
Manager, and Protect. Five sandbox surfaces, two MCP tools, one QuickJS
WASM sandbox. Cloudflare's Code-Mode pattern applied to UniFi's 70+
endpoints — your LLM context stays small no matter how big the API gets.

Single-user (env vars) and multi-user (per-request HTTP headers) modes.
Stdio + Streamable HTTP transports. Cloudflare Workers entry. Strict TLS
by default with per-tenant overrides.

## What you can do

```js
// One key, every console you manage. No controller exposure.
const net = unifi.cloud.network('CONSOLE_ID');
const sites = net.sites.listSites();

// Same shape, Protect.
const p = unifi.cloud.protect('CONSOLE_ID');
const cameras = p.request({ method: 'GET', path: '/v1/cameras' });

// Or stay on the LAN.
const local = unifi.local.devices.listAdoptedDevices({ siteId: 'default' });
```

The full surface is documented in [`SKILL.md`](../../SKILL.md) (the
operating manual the LLM reads) and [`docs/usage.md`](../usage.md) (for
humans). To make an LLM act as a UniFi expert against this server, see
[`examples/unifi-expert-agent/`](../../examples/unifi-expert-agent/).

## What's wired but NOT yet verified

We want bug reports for these in particular:

- **Direct local Protect** (`unifi.local.protect.*`). Same `HttpClient`
  shape as the cloud-proxied variant we verified, but no LAN smoke yet.
- **Mutation Protect operations** — PTZ commands, disable-mic, alarm-
  manager webhook trigger.
- **Binary / streaming Protect surfaces** — snapshot bytes, RTSPS
  metadata, talk-back, `subscribe/*` WebSockets. Not implemented yet
  (the JSON-only `HttpClient` doesn't speak them).
- **Other agent platforms** — Claude Code, Claude Desktop, VS Code +
  GitHub Copilot, Codex CLI, Continue, Cline, MCP Inspector, Aider,
  Zed. Install snippets are in
  [`examples/unifi-expert-agent/install.md`](../../examples/unifi-expert-agent/install.md)
  but each one is **NOT VERIFIED** by us — please file a verification
  report (template in `.github/ISSUE_TEMPLATE/verification_report.yml`).
- **Cursor IDE chat panel** specifically — tested via `cursor-agent`
  CLI but not in the IDE's chat panel.
- **Hosted multi-tenant deployment** behind a reverse proxy.
- **Long-running soak / stability** under sustained load.
- **Real UniFi networks other than the maintainer's homelab.** A single
  network is not generalisable evidence.

See the README's [verification status](../../README.md#verification-status)
table for the full matrix.

## Highlights since the initial scaffold

- **Two new sandbox surfaces:** `unifi.local.protect.*` and
  `unifi.cloud.protect(consoleId).*`. Loader auto-fetches Ubiquiti's
  official Protect OpenAPI spec; bundled curated fallback (~18
  JSON-over-HTTP ops) ships as a CDN-unreachable backstop.
- **Smarter `synthesizeOperationId`** — the official Protect spec ships
  every operation with `operationId: null`. The synthesizer now
  recognises five common REST shapes and produces friendly names like
  `listCameras`, `cameraPtzPatrolStart`, `getCameraSnapshot` instead of
  the legacy `getV1CamerasIdPtzPatrolStartSlot` form.
- **Three live bug fixes** caught by the cloud-Protect verification:
  proxy gating (cloud-Protect was missing whenever cloudSpec was
  absent), Protect version-discovery URL (was probing the wrong path
  for cloud baseUrls), error-namespace label (every miss reported as
  `unifi.local`).
- **Documentation correction:** the initial Protect implementation
  documented the cloud-Protect path as "UNVERIFIED — Ubiquiti has not
  publicly documented" the connector proxying Protect. That was wrong:
  `developer.ui.com/protect/v7.0.107/gettingstarted` exposes a
  "Remote" / "Local" base-URL selector mapping every operation to
  exactly the paths this server emits. All references corrected.

## Install

```bash
git clone https://github.com/jmpijll/unifi-code-mode-mcp.git
cd unifi-code-mode-mcp
npm install
cp .env.example .env  # set UNIFI_LOCAL_API_KEY and/or UNIFI_CLOUD_API_KEY
npm run build
npm start
```

For the multi-tenant HTTP transport: `MCP_TRANSPORT=http npm start` and
pass `X-Unifi-*` headers per request.

For agent-specific install snippets (Cursor, opencode, Claude Code,
Claude Desktop, VS Code Copilot, …) and a ready-made "UniFi expert
agent" persona, see
[`examples/unifi-expert-agent/`](../../examples/unifi-expert-agent/).
