NOTABLE LESSONS FOR DOCS
  * SKILL.md §1 says the execute sandbox is "Async (host calls appear
    sync)". Both Sonnet 4.6 (in earlier verifications) and DeepSeek
    v4 Flash interpret "appear sync" inconsistently — DeepSeek tried
    top-level `await` THREE times before figuring out the IIFE
    pattern. Consider explicitly stating in SKILL.md and in the
    `execute` tool description that:
       - top-level return is NOT allowed (use an expression statement)
       - top-level await is NOT allowed (host calls already block
         synchronously inside QuickJS — no await needed)
       - the LAST expression's value is what's returned to the caller
    This won't catch every model but should reduce the wrong-attempt
    count materially.

  * The unifi.<surface>.unknown-operation error class works as
    designed — DeepSeek read "No operation 'listSites'" and
    immediately knew to call unifi_search. This validates the
    [unifi.<surface>.<error-class>] error-shape contract documented
    in AGENTS.md §6 from a real-LLM perspective.

  * opencode startup time on a cold cache (no pre-existing process
    pool) is ~7 minutes for the FIRST `opencode run` invocation. This
    is opencode-internal (plugin init, project scan, snapshot), not
    related to our MCP server. Subsequent runs in the same session
    are fast (<2 s startup).
