WHAT THIS PROVES
  ✓ The Site Manager → Protect cloud-connector path
        api.ui.com/v1/connector/consoles/{id}/proxy/protect/integration/...
    works against real Protect-enabled hardware with a Site Manager API key.
    The URL pattern matches Ubiquiti's developer.ui.com docs exactly
    (https://developer.ui.com/protect/v7.0.107/gettingstarted, "Remote"
    base-URL selector).
  ✓ The cloud-Protect proxy factory `unifi.cloud.protect(consoleId).*`
    exposed by the QuickJS sandbox is wired correctly: it loads the
    official OpenAPI spec from apidoc-cdn.ui.com (35 ops resolved),
    builds the per-console proxy on demand, dispatches both raw
    request() and operation lookups through the right
    /v1/connector/consoles/{id}/proxy/protect/integration prefix.
  ✓ Two earlier-uncovered bugs are fixed and stay fixed:
      1. cloud.protect(consoleId) was previously gated on having a Site
         Manager (cloudSpec) loaded — meaning Protect-only deployments
         silently lost the cloud-Protect surface.
      2. The Protect spec loader probed
         api.ui.com/proxy/protect/integration/v1/meta/info for version
         discovery — wrong path for the cloud connector. Now skipped
         when baseUrl is api.ui.com; falls through cleanly to the
         known-version ladder.

WHAT THIS DOES NOT PROVE
  ✗ Direct local Protect path /proxy/protect/integration/v1/... — we
    only exercised the cloud-connector path. The local code path is the
    same shape (HttpClient + path prefix), but no LAN-side smoke yet.
  ✗ Mutation operations (PTZ goto, patrol start/stop, disable mic).
    This run is purely read-only: GET /v1/meta/info and GET /v1/cameras.
  ✗ Binary/streaming operations (snapshot, RTSPS, talkback,
    subscribe/* WebSockets) — all out of scope for the JSON-only
    HttpClient.
  ✗ End-to-end LLM-mediated invocation. This was driven directly from
    a Node script through ExecuteExecutor — a real client (cursor-agent,
    opencode, etc.) issuing the `execute` tool with the same code has
    not been exercised against Protect yet.

EXACT SANDBOX CODE EXECUTED
  var p = unifi.cloud.protect("<consoleId>");
  var meta = p.request({ method: 'GET', path: '/v1/meta/info' });
  var cameras = [];
  try {
    var camRes = p.request({ method: 'GET', path: '/v1/cameras' });
    cameras = Array.isArray(camRes) ? camRes
            : (camRes && camRes.data) ? camRes.data : [];
  } catch (e) { cameras = ['ERROR: ' + String(e)]; }
  ({
    meta: meta,
    cameraCount: cameras.length,
    cameraSample: cameras.slice(0, 5).map(function(c) {
      return c && typeof c === 'object'
        ? { id: c.id, name: c.name, state: c.state, type: c.type }
        : { raw: c };
    })
  });
